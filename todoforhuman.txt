# EssKay Sportswear - Important Instructions for Human

## 1. Firebase Rules Update (CRITICAL)
Your current rules might be blocking the Admin Panel from loading the user list. Please update your Firebase Realtime Database rules to the following:

```json
{
  "rules": {
    "users": {
      // Allow the admin to see the full list of users
      ".read": "root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'",
      "$uid": {
        // Allow users to see their own profile
        ".read": "$uid === auth.uid",
        // Allow users to edit their own profile OR admin to manage it
        ".write": "$uid === auth.uid || root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'"
      }
    },
    "products": {
      ".read": "true",
      ".write": "root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'"
    },
    "carts": {
      "$uid": {
        ".read": "$uid === auth.uid",
        ".write": "$uid === auth.uid"
      }
    },
    "customOrders": {
      ".indexOn": ["userId"],
      // Allow admin to see all custom orders
      ".read": "root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'",
      "$orderId": {
        ".write": "auth != null",
        ".read": "data.child('userId').val() === auth.uid || root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'"
      }
    },
    "standardOrders": {
      ".indexOn": ["userId"],
      // Allow admin to see all standard orders
      ".read": "root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'",
      "$orderId": {
        ".write": "auth != null",
        ".read": "data.child('userId').val() === auth.uid || root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'"
      }
    },
    "orders": {
      // Allow admin to see all orders
      ".read": "root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'",
      "$orderId": {
        ".write": "auth != null",
        ".read": "data.child('userId').val() === auth.uid || root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'"
      }
    }
  }
}
```

## 2. Why "Error showing users" happens?
This error occurs because the Admin Panel is trying to fetch the entire list of users from the `users` folder in your database. If the security rules (above) are not set to allow the admin (`business.esskaysportswear@gmail.com`) to read the parent folder, Firebase will block the request.

## 3. Admin Panel Tips
- **Reset Password**: This sends an official Firebase email to the user.
- **Disable User**: Setting a user to "Disabled" will prevent them from adding items to their cart or placing orders (if you implement the check on the frontend).
- **Edit Products**: Remember to fill in the **Category** and check the **Discount** box if you want to show strikethrough pricing.

## 4. Assets
- Make sure icons like `homen.png`, `productsn.png`, `cartn.png` are in the root folder.
- Cloudinary preset `esskay_unsigned` must be active.
