# FIREBASE SETUP GUIDE

### 1. Database URL Debug
- I have added the Database URL to the error message.
- If you see "No account found... DB: Default", it means the `databaseURL` in `firebase.js` is NOT being picked up.
- If you see "DB: https://...", verify that this URL is EXACTLY the same as the one in your Firebase Console.

### 2. Check Database Structure
- Your data MUST look exactly like this in the Firebase Console:
  ```
  esskaysportsdata-default-rtdb
   └── users
        └── xyz123 (This must be the UID)
             ├── email: "user@gmail.com"
             └── name: "User Name"
  ```
- If your users are NOT inside a folder named `users`, or if the ID is NOT the UID, the login will fail.

### 3. Rules Update (CRITICAL)
- To fix "PERMISSION_DENIED", copy and paste these rules into your **Firebase Console > Realtime Database > Rules** tab:
```json
{
  "rules": {
    "users": {
      "$uid": {
        ".read": "auth != null",
        ".write": "auth != null && auth.uid === $uid"
      }
    },
    "customOrders": {
      ".read": "auth != null && root.child('users').child(auth.uid).child('email').val() === 'business.esskaysportswear@gmail.com'",
      ".write": "auth != null"
    }
  }
}
```
*Note: The `customOrders` write permission allows any logged-in user to place an order, but only the admin can read them.*

### 4. Image Size Limit
- If you still get an error, ensure your images are **small (under 500KB)**. Large images can exceed the database's single-node limit.

### 5. Deployment Note
- **Ctrl + F5** to hard refresh after updating rules.
- Check the console (F12) for any specific errors.
